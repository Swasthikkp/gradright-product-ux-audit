# 📖 Overview

This repository contains a comprehensive **Product UX/UI Audit** conducted on the **GradRight** mobile application during a Product Talent Hunt assignment.

Instead of simply identifying bugs, this audit evaluates the application from a **Product Analyst's perspective**, focusing on user journeys, navigation, usability, interface consistency, and overall product experience.

Every issue has been documented with supporting evidence, clear reproduction steps, and actionable recommendations that can be immediately used by Product Managers, Designers, QA Engineers, and Developers.

---

# 🎯 Objective

Evaluate the GradRight mobile application from a **first-time user's perspective** and identify opportunities to improve:

- User Experience (UX)
- Product Usability
- Navigation Flow
- Information Architecture
- UI Consistency
- Accessibility
- Product Quality
- Feature Discoverability

# 🚨 Top 3 Critical Product Findings

| Priority | Issue | Impact |
|----------|-------|--------|
| 🔴 Critical | App exits completely on swipe-back gesture from On-Campus page | Breaks expected navigation behaviour and causes immediate user drop-off |
| 🔴 Critical | Profile menu creates navigation dead-ends on Community page | Prevents users from continuing their journey |
| 🔴 Critical | Community lacks moderation and self-management features | Creates trust, safety, and platform credibility risks |
